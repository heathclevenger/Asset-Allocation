Overall Allocation

Explanation

A high-level summary of how the target-volatility allocation model works.

What The Model Is Trying To Do

The model builds five portfolios from Conservative through Aggressive Growth. Each portfolio is designed around a target volatility range, meaning the model first asks how much risk the portfolio should take, then searches for an allocation that fits that risk level while trying to improve expected return.

Inputs

The model uses compound return, volatility, and correlation assumptions from the uploaded quarterly capital market assumptions file. Cash is fixed at 1% in every portfolio. Asset class minimums and maximums set the guardrails for how much the optimizer is allowed to use in each sleeve.

Portfolio Profiles

The Moderate portfolio is the anchor for volatility. When the Moderate volatility target is changed, the other four portfolios move up or down with it using fixed relative adjustments. Category ranges for Equity, Fixed Income, Alternatives, and Cash still apply separately to each profile.

Optimization Process

The app runs two allocation tests. The first is a target-volatility optimization that looks for the best return within the allowed risk and allocation constraints. The second is a Monte Carlo search that tests many possible portfolios and keeps the strongest fit. The main output averages those two results.

Sub-Sleeve Allocation

After the model decides the broad category weights, each category is spread across its underlying sleeves using the current pro-rata mix. This keeps the sub-sleeve allocations aligned with the overall category totals, so the detailed weights add back to the main portfolio allocation.

How To Read The Output

The Dashboard shows the final sleeve weights for each portfolio. Model Summary shows expected return, volatility, status versus the target range, and broad category weights. If a portfolio is below or above target, the constraints may be preventing the model from reaching the desired volatility range.
